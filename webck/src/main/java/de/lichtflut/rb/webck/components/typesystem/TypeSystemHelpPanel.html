Type System - View, Edit, Create

In diesem Bereich kann das interne "Type System" von Glasnost eingesehen und angepasst werden. Es besteht im wesentlichen aus drei Elementen.

  1. Typen (mit ihrem Schema):

    Diese klassifizieren und typisieren die Entitäten, die in Glasnost verwalter werden können. Jede Entität hat (mindestens) einen Typ. Zu jedem Typ kann ein Schema definiert werden, das vorgibt, welche Attribute und Relationen Entitäen dieses Typs i.d.R. haben. Ein Schema besteht aus einer Liste an Property Deklarationen.

  2. Properties:

    Durch diese "Eigenschafts-Typen" wird die Bedeutung eines Attributs oder einer Relation festgelegt. Einer Entität wird nicht einfach ein Wert oder eine Relation zu einer anderen Entität zugeordnet, sondern diese Zuweisung wird durch eine Property charakterisiert.

  3. Public Type Definitions:

    Jede Property Deklaration hat eine Typ-Definition. Hier wird festgelegt, um welchen Datentypen es geht (Ganzzahl, Zeichenkette, Resourcen-Referenz) und welche Constraints dafür gelten. Es ist auch möglich Allgemeine Typ Definitionen zu erstellen, die aus mehreren Property Deklarationen wiederverwendet werden können.
